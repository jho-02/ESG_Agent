# Week 13 - Lean Startup 실험 운영

이 문서는 11주차 Feature Flag 및 A/B 테스트 구조, 12주차 테스트 자동화 결과를 바탕으로 Lean Startup 실험을 2주 운영했다고 가정한 mock 실험 문서다.
실제 사용자 대상 운영 결과가 아니라, LLM 기반 가상 페르소나와 mock 실험 로그를 사용한 과제 제출용 문서다.
공개 저장소이므로 실제 데이터, API Key, DB URL, 고객사 정보, 실제 사용자 개인정보는 포함하지 않는다.

## 실험 목표

- Feature Flag 기반 UI 실험의 반응을 확인한다.
- Variant A와 Variant B 중 어떤 화면 구성이 더 이해하기 쉬운지 비교한다.
- 챗봇 패널, 근거자료 다운로드, 요약 중심 UI의 선호도를 mock 방식으로 수집한다.

## 실험 가설

- Variant B는 요약 리포트 강조형 UI이므로 첫 이해 시간이 짧을 것이다.
- Variant A는 기존 대시보드 중심 UI라서 상세 지표 비교에는 더 익숙할 것이다.
- 챗봇 패널과 근거자료 다운로드는 일부 사용자에게서 높은 가치가 확인될 것이다.

## 실험 구조

- 실험 기간: 2주 가정
- 실험 대상: LLM 기반 가상 페르소나 10명
- 실험 방식: Feature Flag 기반 A/B 테스트
- 측정 범위: 대시보드 첫 이해 시간, 근거자료 다운로드 클릭률, 챗봇 패널 사용률, 리포트 요약 영역 클릭률, 사용자 만족도 mock score

## 결과 요약

- Variant B는 요약 중심 이해 속도와 만족도에서 우세한 mock 결과를 보였다.
- Variant A는 상세 지표 비교 상황에서 선호도가 유지되었다.
- 페르소나 피드백에서는 초보 사용자와 모바일 가독성 중시 사용자가 Variant B를 더 선호했다.
- 전체적으로는 기존 구조를 유지하되, 일부 UI 개선을 backlog로 넘기는 방향이 적절하다고 판단했다.

## 결정 방향

- 기본 방향은 Persevere이다.
- 대시보드의 핵심 구조는 유지한다.
- 요약 리포트 강조형 UI 요소는 별도 backlog로 분리해 점진 반영을 검토한다.

## 실험 로그 해석

이 문서의 수치는 실제 운영 데이터가 아니라 mock 실험 결과다.
실제 운영 반영 전에는 추가 검증이 필요하다.

## 자동화 리포팅 관련 메모

GitHub Issues, Projects, Actions 기반 자동 주간 리포팅은 선택과제이므로 이번 범위에서는 실제 구현하지 않는다.
향후 확장 계획으로만 검토한다.
