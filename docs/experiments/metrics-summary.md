# 핵심 지표 변화 요약

이 문서는 13주차 Lean Startup 실험의 핵심 지표를 비교한 mock 측정표다.
수치는 과제용 mock 값이며 실제 운영 데이터가 아니다.

## 비교 기준

- 비교 방식: Variant A 대비 Variant B
- 해석 기준: 더 낮은 시간, 더 높은 클릭률, 더 높은 만족도가 긍정적

## 지표 변화 표

| 지표 | Variant A | Variant B | 개선률 | 해석 |
|---|---:|---:|---:|---|
| 대시보드 첫 이해 시간(초) | 42 | 31 | 26.2% 개선 | B가 더 빠름 |
| 근거자료 다운로드 클릭률 | 38% | 34% | 10.5% 감소 | A가 근거 확인에는 유리 |
| 챗봇 패널 사용률 | 21% | 29% | 38.1% 개선 | B에서 탐색 상호작용 증가 |
| 리포트 요약 영역 클릭률 | 27% | 46% | 70.4% 개선 | B가 요약 관심을 더 끔 |
| 사용자 만족도 mock score | 3.8 | 4.3 | 13.2% 개선 | B가 전반적으로 높음 |

## 해석 요약

- 요약 중심 UI는 이해 속도와 만족도 측면에서 개선 효과가 컸다.
- 기존 대시보드 중심 UI는 근거자료 다운로드와 상세 비교에서 여전히 필요했다.
- 따라서 화면 구조를 완전히 바꾸기보다는, 요약 중심 요소를 보강하는 방향이 적절하다.

## 테스트 자동화와의 연결

12주차 coverage/test 결과로 실험 관련 순수 함수와 mock 추적 구조의 안정성을 검증했다.
즉, 실험 로직의 안정성은 테스트 자동화로 검증했고, 이 문서의 지표는 그 위에서 해석한 mock 결과다.

## 수동 확인 필요

- 실제 운영 환경의 수치와는 다를 수 있다.
- 실제 사용자 데이터가 들어가면 재측정이 필요하다.
- 커버리지 수치와 실험 결과 해석은 제출 시점 기준으로 업데이트할 수 있다.
