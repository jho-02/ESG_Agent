# A/B 테스트 결과 요약

이 문서는 2주 운영을 가정한 mock A/B 테스트 결과다.
실제 운영 결과가 아니라, 11주차 Feature Flag 및 A/B 테스트 구조를 기준으로 만든 과제 제출용 mock 로그다.

## 실험 개요

| 항목 | 내용 |
|---|---|
| 실험명 | week13-lean-startup-ab-test |
| 기간 | 2주 가정 |
| 대상 | LLM 기반 가상 페르소나 10명 |
| Variant A | 기존 대시보드 중심 UI |
| Variant B | 요약 리포트 강조형 UI |

## 지표 정의

- 대시보드 첫 이해 시간: 첫 화면을 읽고 핵심을 파악하는 데 걸린 시간
- 근거자료 다운로드 클릭률: 다운로드 버튼 클릭 비율
- 챗봇 패널 사용률: 질문 입력 또는 패널 열기 비율
- 리포트 요약 영역 클릭률: 요약 카드 또는 요약 섹션 클릭 비율
- 사용자 만족도 mock score: 5점 만점 가상 만족도 점수

## mock 결과

| 지표 | Variant A | Variant B | 해석 |
|---|---:|---:|---|
| 대시보드 첫 이해 시간(초) | 42 | 31 | B가 더 빠름 |
| 근거자료 다운로드 클릭률 | 38% | 34% | A가 소폭 우세 |
| 챗봇 패널 사용률 | 21% | 29% | B가 더 활발함 |
| 리포트 요약 영역 클릭률 | 27% | 46% | B가 명확히 우세 |
| 사용자 만족도 mock score | 3.8 / 5 | 4.3 / 5 | B가 우세 |

## 2주 운영 흐름

### 1주차

- persona-01, persona-03, persona-04, persona-07, persona-09는 Variant B를 선호했다.
- persona-02, persona-05, persona-08, persona-10은 Variant A를 유지하는 편이 낫다고 응답했다.

### 2주차

- 요약 리포트 강조형 UI에 대한 긍정 피드백이 더 많이 누적됐다.
- 상세 비교를 중시하는 사용자에서는 기존 대시보드의 안정성이 확인됐다.
- 챗봇 패널은 특정 페르소나에서만 강한 선호를 보였다.

## 해석 메모

- Variant B는 첫 이해 속도와 만족도에서 유리했다.
- Variant A는 근거자료와 상세 비교 문맥에서 여전히 필요했다.
- 실제 운영 데이터가 아니라 mock 실험 결과이므로, 결과는 제출용 참고 자료로만 사용한다.
