# 가상 페르소나 피드백

이 문서는 LLM 기반 가상 페르소나 10명의 mock 피드백을 정리한 문서다.
실제 사용자, 실제 사용자 ID, 실제 개인정보는 포함하지 않는다.

## persona-01

- 역할 또는 사용 맥락: ESG 평가 초보 사용자
- 주요 사용 목적: 전체 흐름을 빠르게 이해하고 싶음
- 사용 패턴: 처음 접속 후 요약부터 확인
- 불편 사항: 세부 지표가 많아 처음에는 복잡하게 느껴짐
- 긍정 피드백: 요약 리포트가 있으면 진입 장벽이 낮아짐
- 개선 요청: 핵심 요약 카드가 더 눈에 띄면 좋겠음
- 선호 variant: B

## persona-02

- 역할 또는 사용 맥락: 기업 재무 담당자 관점 사용자
- 주요 사용 목적: 재무와 ESG 연결 관점 확인
- 사용 패턴: 상세 지표와 근거자료를 함께 살펴봄
- 불편 사항: 다운로드 위치를 찾는 데 시간이 걸림
- 긍정 피드백: 대시보드 구조는 익숙하고 비교가 쉬움
- 개선 요청: 근거자료 버튼을 더 분명하게 배치하면 좋겠음
- 선호 variant: A

## persona-03

- 역할 또는 사용 맥락: 투자자 관점 사용자
- 주요 사용 목적: 빠르게 리스크와 강점을 파악
- 사용 패턴: 첫 화면에서 핵심 수치만 먼저 읽음
- 불편 사항: 초기 정보가 분산되면 판단이 늦어짐
- 긍정 피드백: 요약 중심 UI가 의사결정에 도움이 됨
- 개선 요청: 핵심 지표를 한 화면에서 더 압축해 보여주면 좋겠음
- 선호 variant: B

## persona-04

- 역할 또는 사용 맥락: 리포트 요약을 선호하는 사용자
- 주요 사용 목적: 긴 분석 대신 핵심 결론 확인
- 사용 패턴: 제목, 요약, 결론 순으로 읽음
- 불편 사항: 대시보드 요소가 많으면 집중이 분산됨
- 긍정 피드백: 리포트 요약 강조가 명확해서 좋음
- 개선 요청: 요약 영역을 더 위로 올리면 좋겠음
- 선호 variant: B

## persona-05

- 역할 또는 사용 맥락: 근거자료 다운로드를 중시하는 사용자
- 주요 사용 목적: 검토 자료를 저장하고 재확인
- 사용 패턴: 화면 확인 후 바로 자료 다운로드 여부를 찾음
- 불편 사항: 다운로드 관련 안내가 눈에 덜 띔
- 긍정 피드백: 근거자료 영역이 있으면 신뢰도가 올라감
- 개선 요청: 다운로드 버튼 위치를 고정하면 좋겠음
- 선호 variant: A

## persona-06

- 역할 또는 사용 맥락: 챗봇 질의를 자주 사용하는 사용자
- 주요 사용 목적: 질문 기반으로 빠른 해석 요청
- 사용 패턴: 지표를 본 뒤 바로 추가 질문을 입력
- 불편 사항: 챗봇 진입점이 약하면 사용 빈도가 줄어듦
- 긍정 피드백: 질문-응답 흐름이 있으면 탐색 효율이 높아짐
- 개선 요청: 챗봇 패널을 더 쉽게 보이게 하면 좋겠음
- 선호 variant: B

## persona-07

- 역할 또는 사용 맥락: 모바일 화면 가독성을 중시하는 사용자
- 주요 사용 목적: 작은 화면에서 빠르게 확인
- 사용 패턴: 스크롤을 적게 하고 요약 먼저 확인
- 불편 사항: 상세 카드가 많으면 가독성이 떨어짐
- 긍정 피드백: 간결한 요약형 UI가 모바일에서 편함
- 개선 요청: 카드 간 간격과 타이포를 더 단순화하면 좋겠음
- 선호 variant: B

## persona-08

- 역할 또는 사용 맥락: 데이터 신뢰성을 중시하는 사용자
- 주요 사용 목적: 수치 출처와 근거를 확인
- 사용 패턴: 수치와 근거자료를 번갈아 확인
- 불편 사항: 근거가 분리되어 있으면 신뢰 판단이 늦어짐
- 긍정 피드백: 근거자료 연결이 있으면 신뢰도가 높아짐
- 개선 요청: 근거자료 링크 안내를 더 분명히 해주면 좋겠음
- 선호 variant: A

## persona-09

- 역할 또는 사용 맥락: UI 단순성을 선호하는 사용자
- 주요 사용 목적: 복잡하지 않은 화면 흐름 확인
- 사용 패턴: 최소한의 메뉴와 핵심 정보만 봄
- 불편 사항: 정보 밀도가 높으면 부담을 느낌
- 긍정 피드백: 요약 리포트 중심 UI가 읽기 쉬움
- 개선 요청: 불필요한 시각 요소를 줄이면 좋겠음
- 선호 variant: B

## persona-10

- 역할 또는 사용 맥락: 상세 지표 비교를 원하는 사용자
- 주요 사용 목적: 항목별 수치 비교와 차이 파악
- 사용 패턴: 카테고리와 세부 지표를 반복적으로 비교
- 불편 사항: 요약만 강하면 비교 정보가 부족할 수 있음
- 긍정 피드백: 기존 대시보드 중심 UI가 비교에 익숙함
- 개선 요청: 비교 테이블과 정렬 옵션이 더 있으면 좋겠음
- 선호 variant: A
