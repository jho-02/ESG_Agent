# Pivot 또는 Persevere 판단

이 문서는 13주차 Lean Startup 실험 결과를 바탕으로 한 mock 판단 문서다.
결론은 Persevere를 기본으로 하되, 일부 UI 개선은 별도 backlog로 넘기는 방향이다.
실제 운영 반영 전에는 추가 검증이 필요하다.

## 판단 요약

- 기본 결정: Persevere
- 이유: Variant B가 요약 이해 속도와 만족도에서 우세했지만, Variant A의 상세 비교와 근거자료 활용도도 유지되었다.
- 해석: 현재 구조를 완전히 뒤집기보다, 요약 리포트 강조 요소를 점진 반영하는 편이 적절하다.

## 결정 근거

1. A/B 테스트 결과에서 Variant B가 핵심 이해 지표를 개선했다.
2. 페르소나 피드백에서 초보 사용자, 모바일 사용자, 요약 선호 사용자의 응답이 긍정적이었다.
3. 상세 지표 비교와 근거자료 다운로드는 Variant A가 여전히 강점으로 남았다.
4. 12주차 테스트 자동화로 실험 로직의 기본 안정성을 확인했다.

## 향후 backlog 항목

- 요약 리포트 카드의 시각적 우선순위 강화
- 근거자료 다운로드 버튼의 가시성 보완
- 챗봇 패널 접근성 개선
- 모바일 화면에서의 카드 간격과 타이포 조정
- 상세 비교 화면의 정렬 및 비교 옵션 보강

## 확장 계획 메모

- 자동화된 주간 리포팅은 선택과제로서 향후 확장 가능하다.
- GitHub Issues, Projects, Actions 기반 리포팅은 이번 범위에서 실제 구현하지 않았다.
- 다음 운영 단계에서는 실제 사용자 대신 더 넓은 mock 페르소나 집합으로 검증을 확장할 수 있다.
